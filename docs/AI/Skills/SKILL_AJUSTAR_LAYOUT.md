# Skill: Ajustar layout

**Entrada:** resultado visual esperado, estados e padrões de interface. **Saída:** interface consistente e verificada.

1. Inspecionar componente, estilos, comportamento e padrões existentes.
2. Definir estados normal, vazio, carregando, erro e responsivo aplicáveis.
3. Reutilizar componentes e tokens antes de criar variações.
4. Preservar semântica, teclado, foco, contraste e mensagens acessíveis.
5. Manter comportamento fora da apresentação quando a arquitetura assim exigir.
6. Validar visualmente nos tamanhos relevantes e executar testes aplicáveis.
7. Atualizar padrão apenas quando surgir convenção reutilizável.

Não misture redesenho amplo com ajuste localizado sem autorização.
