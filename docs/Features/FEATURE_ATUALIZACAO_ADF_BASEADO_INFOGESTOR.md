# FEATURE - Atualização do AI Development Framework (ADF) baseada na evolução do InfoGestor

## Objetivo

Atualizar o projeto **AI Development Framework (ADF)** localizado em:

``` text
D:\RepositorioGit\TrollComS\AI-Development-Framework
```

utilizando como referência a evolução arquitetural realizada no projeto:

``` text
D:\RepositorioGit\InfoGestor
```

O objetivo **não é copiar o InfoGestor**, mas sim incorporar ao ADF
todas as melhorias que se mostraram genéricas, reutilizáveis e
aplicáveis a qualquer projeto.

------------------------------------------------------------------------

# Objetivos da análise

Realizar uma análise completa entre o estado atual do ADF e a
arquitetura/documentação utilizada no InfoGestor.

Identificar:

-   melhorias estruturais;
-   novos padrões;
-   simplificações;
-   organização documental;
-   melhorias no fluxo de desenvolvimento;
-   melhorias na comunicação entre IA Arquiteta, IA Executora e IA
    Revisora;
-   melhorias no ADF descobertas durante o desenvolvimento do
    InfoGestor.

Tudo que for específico do InfoGestor NÃO deve ser levado para o ADF.

------------------------------------------------------------------------

# O que deve ser analisado

Analisar toda a estrutura documental do InfoGestor.

Principalmente:

-   docs/AI
-   docs/Arquitetura
-   docs/Padroes
-   docs/Templates
-   docs/Projeto

Verificar se existe alguma evolução em:

-   organização
-   nomenclatura
-   responsabilidades
-   separação de documentos
-   padronização
-   fluxo de leitura
-   redução de contexto
-   redução de consumo de tokens
-   facilidade de manutenção

------------------------------------------------------------------------

# Índice principal

Verificar se o modelo atual do ADF pode ser melhorado utilizando a ideia
do:

``` text
INDICE_DOCUMENTACAO.md
```

como ponto único de entrada da documentação.

Caso existam melhorias realizadas no InfoGestor, replicá-las para o ADF
de forma genérica.

------------------------------------------------------------------------

# Skills

Reavaliar todas as Skills.

Verificar se durante o desenvolvimento do InfoGestor surgiram padrões
melhores para:

-   criação de Features
-   revisão
-   arquitetura
-   integração externa
-   CRUD
-   correção de bugs
-   ajuste de layout
-   refatoração

Caso existam melhorias, atualizar as Skills do ADF.

------------------------------------------------------------------------

# Templates

Comparar todos os Templates.

Verificar se existe oportunidade de simplificação.

Padronizar:

-   Feature
-   Revisão
-   Prompt
-   Checklist
-   Decisão Arquitetural

Sempre buscando diminuir repetição de conteúdo.

------------------------------------------------------------------------

# Organização da documentação

Reavaliar a organização dos diretórios.

Verificar se a estrutura atual continua sendo a melhor opção.

Caso exista uma estrutura mais limpa utilizada no InfoGestor, aplicar no
ADF.

------------------------------------------------------------------------

# Fluxo de leitura

Validar se o fluxo atual continua adequado.

Garantir que exista claramente:

-   ponto de entrada;
-   sequência mínima de leitura;
-   documentação obrigatória;
-   documentação complementar.

------------------------------------------------------------------------

# Padrões

Comparar os padrões existentes.

Verificar se durante o desenvolvimento do InfoGestor foram criados novos
padrões genéricos.

Exemplos:

-   organização documental;
-   nomenclatura;
-   responsabilidades;
-   separação entre arquitetura e negócio;
-   separação entre padrões e templates;
-   fluxo de atualização documental.

Adicionar ao ADF apenas aquilo que possa ser utilizado em qualquer
projeto.

------------------------------------------------------------------------

# Redução de tokens

Reavaliar toda a documentação.

Sempre que possível:

-   eliminar repetições;
-   consolidar informações;
-   remover redundâncias;
-   reduzir contexto desnecessário;
-   melhorar referências cruzadas;
-   melhorar reutilização.

Objetivo:

Diminuir consumo de tokens sem perda de qualidade.

------------------------------------------------------------------------

# Documentação duplicada

Procurar documentos com responsabilidades sobrepostas.

Caso existam:

-   consolidar;
-   reorganizar;
-   criar referências cruzadas;
-   evitar duplicidade.

------------------------------------------------------------------------

# IA Arquiteta

Verificar se as responsabilidades continuam adequadas.

Caso o InfoGestor tenha evoluído esse fluxo, aplicar ao ADF.

------------------------------------------------------------------------

# IA Executora

Reavaliar o fluxo mínimo necessário para execução.

Objetivo:

reduzir leitura desnecessária.

------------------------------------------------------------------------

# IA Revisora

Reavaliar checklist.

Melhorar:

-   validação arquitetural;
-   validação documental;
-   validação de padrões;
-   validação de testes.

------------------------------------------------------------------------

# Atualização automática da documentação

Caso existam melhorias implementadas no InfoGestor relacionadas à
atualização documental após uma Feature, incorporar ao ADF.

------------------------------------------------------------------------

# Compatibilidade

Garantir que o ADF continue independente de qualquer projeto específico.

Nenhum documento poderá conter:

-   InfoGestor;
-   Asaas;
-   Empresa;
-   Cobrança;
-   Cliente;
-   qualquer regra de negócio específica.

Todo o conteúdo deve permanecer genérico.

------------------------------------------------------------------------

# Resultado esperado

Ao término da análise:

-   atualizar documentos existentes;
-   reorganizar documentação quando necessário;
-   criar novos documentos caso façam sentido;
-   remover documentos obsoletos;
-   atualizar índices;
-   atualizar mapas;
-   atualizar templates;
-   atualizar Skills;
-   atualizar o fluxo principal do ADF.

------------------------------------------------------------------------

# Restrições

Não alterar a filosofia do ADF.

Manter compatibilidade com projetos já existentes.

Evitar mudanças apenas por preferência.

Toda alteração deve possuir ganho claro em:

-   organização;
-   manutenção;
-   reutilização;
-   clareza;
-   redução de contexto;
-   escalabilidade.

------------------------------------------------------------------------

# Entregáveis

Ao finalizar:

1.  Atualizar toda a documentação do ADF.

2.  Atualizar índices e mapas.

3.  Atualizar Templates.

4.  Atualizar Skills.

5.  Atualizar o fluxo de leitura.

6.  Atualizar o README principal caso necessário.

7.  Gerar um documento:

``` text
CHANGELOG_ADF.md
```

contendo:

-   documentos alterados;
-   documentos criados;
-   documentos removidos;
-   justificativa de cada alteração;
-   benefícios obtidos.

------------------------------------------------------------------------

# Critério de qualidade

Antes de finalizar, realizar uma revisão completa garantindo que:

-   não existam documentos órfãos;
-   não existam links quebrados;
-   não existam referências inválidas;
-   todos os índices apontem corretamente para os novos documentos;
-   toda a documentação permaneça consistente.
